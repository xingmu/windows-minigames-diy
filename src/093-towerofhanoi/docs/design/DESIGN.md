# 汉诺塔游戏设计文档

## 1. 游戏概述

### 游戏类型
经典益智游戏

### 核心玩法
将所有盘子从杆 A 移动到杆 C，使用杆 B 作为辅助杆。

### 目标用户
- 益智游戏爱好者
- 算法学习者
- 休闲玩家

## 2. 技术规格

### 游戏参数
- **默认盘子数**: 5
- **最小盘子数**: 3
- **最大盘子数**: 8
- **杆子数量**: 3
- **窗口尺寸**: 800 x 600 像素
- **目标帧率**: 60 FPS

### 数据结构

```zen-c
// 杆子结构
struct Peg {
    disks: [MAX_DISKS]i32,  // 存储每层盘子的大小
    count: i32              // 盘子在杆子上的数量
}

// 游戏状态
struct GameState {
    pegs: [MAX_PEGS]Peg,     // 三根杆子
    selected_peg: i32,       // 当前选中的杆子
    disk_count: i32,        // 盘子数量
    move_count: i32,        // 移动次数
    game_over: bool,        // 游戏是否胜利
    ...
}
```

## 3. 视觉设计

### 颜色方案
- **背景**: 深蓝色 `#2C3E50`
- **杆子**: 木头棕色 `#8B5A2B`
- **底座**: 深棕色 `#654321`
- **选中杆**: 金色 `#FFD700`
- **盘子**: 彩虹色渐变（红橙黄绿蓝紫等）

### 盘子尺寸计算
```
最小盘宽度 = 50px
最大盘宽度 = 180px
盘子高度 = 30px
宽度增量 = (最大宽度 - 最小宽度) / (盘子数 - 1)
```

## 4. 游戏流程

### 初始化
1. 在杆 A 上按顺序放置所有盘子（大盘在下）
2. 设置当前选中的杆子为 -1（无选中）
3. 重置移动次数为 0
4. 计算目标移动次数 `2^n - 1`

### 游戏循环
1. 等待玩家输入
2. 根据输入选择杆子或移动盘子
3. 更新游戏状态
4. 检查是否胜利
5. 重绘画面

### 胜利条件
杆 C 上的盘子数量等于总盘子数量

## 5. 交互设计

### 输入处理
- 数字键 `1/2/3`: 直接选择对应杆子
- 方向键 `A/D`: 左右选择杆子
- `上箭头/W/空格/回车`: 执行移动
- `R`: 重新开始
- `N/M`: 增删盘子数并重新开始
- `ESC`: 退出游戏

### 移动验证
```
can_move(from, to) {
    if (源杆为空) return false
    if (目标杆已满) return false
    if (目标杆顶部盘子 < 源杆顶部盘子) return false
    return true
}
```

## 7. 实现要点

### 跨平台支持
- 通过条件编译 `#if IS_WINDOWS / IS_LINUX / IS_MACOS` 处理平台差异
- 所有游戏逻辑使用纯 Zen-C 代码
- 平台特定代码放在 `platform_api.zc` 中

### 关键算法
汉诺塔的递归解法：
```
function hanoi(n, from, to, aux):
    if n == 1:
        move disk 1 from from to to
    else:
        hanoi(n-1, from, aux, to)
        move disk n from from to to
        hanoi(n-1, aux, to, from)
```
