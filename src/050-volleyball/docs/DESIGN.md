# Volleyball 游戏设计文档

## 1. 概述

### 项目名称
050-Volleyball（排球）

### 项目类型
2D 体育竞技游戏

### 核心功能
玩家与电脑对战的排球游戏，双方通过控制球拍击球，先得21分获胜。

### 目标平台
Windows GUI

## 2. 技术架构

### 语言与框架

- **框架**: 003-framework
- **图形**: 帧缓冲直接像素绘制

## 3. 游戏设计

### 3.1 画面布局

```
┌──────────────────────────────────┐
│           CPU 分数               │  ← 顶部
│      [红色球拍 - AI控制]         │  ← CPU 球拍区
│ - - - - - 发球线 - - - - - - - - │
│                                  │
│ ════════════ 网 ════════════════ │  ← 球网（中心）
│                                  │
│ - - - - - 发球线 - - - - - - - - │
│    [橙色球拍 - 玩家控制]         │  ← 玩家球拍区
│           玩家分数               │  ← 底部
└──────────────────────────────────┘
```

### 3.2 游戏参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 窗口宽度 | 480px | 游戏窗口宽度 |
| 窗口高度 | 640px | 游戏窗口高度 |
| 目标帧率 | 60 FPS | 游戏刷新率 |
| 球拍宽度 | 100px | 球拍宽度 |
| 球拍高度 | 15px | 球拍高度 |
| 球半径 | 12px | 排球半径 |
| 初始球速 | 6 | 首帧移动像素 |
| 最大球速 | 14 | 最大帧移动像素 |
| 速度增量 | 0.3 | 每次击球后速度增加 |
| 获胜分数 | 21 | 比赛获胜所需分数 |

### 3.3 游戏状态

```
Ready ──[空格/点击]──> Serving ──[空格]──> Playing
                              ↑              │
                              │              ↓
                              └──── GameOver ←┘
```

- **Ready**: 初始状态，显示开始界面
- **Serving**: 发球状态，球跟随发球方
- **Playing**: 比赛中，球在飞行
- **GameOver**: 游戏结束，显示胜负

### 3.4 物理系统

#### 球运动
- 球的位置由 `ballX`, `ballY` 更新
- 每帧更新: `ballX += ballVX`, `ballY += ballVY`
- 初始发球角度: -60° 到 +60° 随机

#### 碰撞检测
- 球与边界: 左右边界反弹
- 球与球拍: 根据击中位置计算反射角度
  - 击中球拍中心: 近似垂直反弹
  - 击中球拍边缘: 倾斜角度更大

#### 速度控制
- 初始速度: 6 像素/帧
- 每次击球后速度增加 0.3
- 最大速度限制: 14 像素/帧

### 3.5 AI 设计

#### CPU 球拍控制
1. 跟踪球当前位置
2. 预测球的落点（简化版：考虑直线轨迹）
3. 平滑移动到目标位置
4. 移动速度限制防止过快

## 4. 交互设计

### 4.1 控制方式

| 输入 | 功能 |
|------|------|
| 空格键 | 开始游戏 / 发球 / 重新开始 |
| 鼠标左键 | 开始游戏 / 发球 / 重新开始 |
| 左方向键 ← | 球拍向左移动 |
| 右方向键 → | 球拍向右移动 |
| 鼠标移动 | 球拍跟随鼠标 X 位置 |
| ESC | 退出游戏 |

### 4.2 游戏流程

1. **启动**: 显示 "按空格键开始发球"
2. **发球**: 玩家按空格，球向斜上方飞出
3. **回合**: 双方交替击球，球速逐渐增加
4. **得分**: 球落到底线对方得分
5. **换发**: 得分方发球
6. **胜利**: 先得21分获胜
