# TetrisAttack 游戏设计文档

## 1. 概述

TetrisAttack（俄罗斯方块消消乐）是一款经典的益智游戏，玩家需要控制下落的方块对，当三个或更多相同颜色的方块相邻时（水平或垂直），它们会自动消除。

## 2. 游戏规格

### 2.1 游戏区域
- **网格尺寸**: 12列 × 18行
- **方块大小**: 28×28 像素
- **方块间距**: 2 像素
- **方块颜色**: 6 种不同颜色

### 2.2 游戏规则
1. 两个方块一起从顶部下落
2. 玩家使用方向键控制方块的位置和交换
3. 当三个或更多相同颜色的方块水平或垂直相连时，自动消除
4. 消除后，上方的方块在重力作用下下落
5. 如果下落后方块仍然相连，会引发连锁消除

### 2.3 计分系统
- **基础分**: 每消除一个方块 +10 分
- **连击奖励**: 连击时额外 +5 分/每连击
- **最终分数**: 所有消除分数的累加

### 2.4 时间限制
- 游戏时间: 3 分钟（180秒）
- 时间归零时游戏结束

## 3. 算法设计

### 3.1 核心机制

```
消除条件：
1. 水平方向：3个或更多相同颜色连续
2. 垂直方向：3个或更多相同颜色连续
```

### 3.2 游戏流程

```
游戏开始:
  1. 初始化游戏网格（底部6行随机填充）
  2. 生成第一个下落方块对

每帧更新:
  1. 处理输入（方向键）
  2. 下落方块自动下落
  3. 检测消除
  4. 应用重力
  5. 重复消除检测（连锁反应）
  6. 生成新的下落方块对

消除检测:
  for each row:
    find horizontal matches (3+ consecutive same color)
  for each column:
    find vertical matches (3+ consecutive same color)
  mark matched blocks for clearing
  clear marked blocks
  apply gravity
  repeat until no more matches
```

### 3.3 重力系统

```
function apply_gravity():
    for each column:
        for row from bottom-1 to top:
            if block[r] is empty and block[r-1] is filled:
                move block[r-1] down to r
                clear block[r-1]
```

## 4. 数据结构

### 4.1 方块结构

```c
struct Block {
    int colorType;    // 颜色类型 (0-5)
    bool visible;     // 是否可见
}
```

### 4.2 下落方块对

```c
struct FallingPair {
    int col;          // 所在列
    int row;          // 上方方块所在行
    int color1;       // 上方圆块颜色
    int color2;       // 下方方块颜色
    bool active;      // 是否正在下落
}
```

### 4.3 游戏状态

```c
struct TetrisAttackState {
    Block[18][12] grid;       // 游戏网格
    FallingPair falling;       // 正在下落的方块对
    int cursorCol;             // 光标列位置
    int score;                  // 当前分数
    int timeLeft;              // 剩余时间
    bool gameOver;            // 游戏结束
    bool gameWon;             // 游戏胜利
    int combo;                 // 连击计数
    int fallSpeed;            // 下落速度（毫秒）
}
```

## 7. 技术实现

### 7.1 渲染方式

使用帧缓冲 (Framebuffer) 直接写入显存：
- 每个像素 32 位 (ARGB)
- Pitch = 宽度 × 4 字节
- 双缓冲消除闪烁

### 7.2 事件处理

- 键盘事件：方向键控制
- 定时器事件：每秒更新时间，自动下落
- 更新事件：定期更新游戏逻辑

### 7.3 游戏循环

```
while running:
    processEvents()     // 处理输入
    update()           // 更新逻辑（下落、消除、重力）
    render()            // 渲染画面
```

## 8. 游戏特性

### 8.1 连击系统
- 当一次消除后引发另一次消除时，连击数增加
- 连击数越高，额外分数越多
- 连击数在无消除时重置

### 8.2 快速下落
- 按下方向键↓可以加速下落
- 快速下落在触底时立即放置方块

### 8.3 颜色交换
- 按下方向键↑可以交换两个下落方块的颜色
- 这个功能对于制造消除至关重要

## 9. 扩展功能

### 9.1 难度递增
- 随着时间推移，下落速度逐渐加快

### 9.2 特殊方块
- 炸弹方块：消除周围方块
- 彩虹方块：可匹配任何颜色

### 9.3 道具系统
- 暂停：暂停游戏计时
- 重置：重新排列所有方块
