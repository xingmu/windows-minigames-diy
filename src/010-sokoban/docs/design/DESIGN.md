# Sokoban 游戏设计文档

## 概述

本文档描述 Zen-C 推箱子（Sokoban）游戏的技术设计和实现细节。

## 游戏规则

推箱子是一种经典的益智游戏，玩家在一个封闭的仓库中推动箱子到指定的目标位置。

### 基本规则

1. 玩家只能推动箱子，不能拉箱子
2. 玩家一次只能推动一个箱子
3. 箱子不能穿过墙壁或其他箱子
4. 当所有箱子都放到目标点上时，关卡完成

### 操作控制

| 按键 | 功能 |
|------|------|
| ← | 向左移动 |
| ↑ | 向上移动 |
| → | 向右移动 |
| ↓ | 向下移动 |
| R | 重置当前关卡 |
| N | 进入下一关（仅在当前关卡完成后）|
| ESC | 退出游戏 |

## 技术架构

### 类结构

```
SokobanGame (extends Game)
├── onInit()           - 初始化回调
├── onShutdown()       - 关闭回调
├── onUpdate(dt)       - 每帧更新
├── onRender(canvas)   - 每帧渲染
├── onKeyDown(key)     - 键盘按下处理
├── sokoban_logic.zc   - 游戏逻辑模块
│   ├── Level          - 关卡数据结构
│   ├── SokobanGameState - 游戏状态
│   ├── 移动逻辑        - 碰撞检测、箱子推动
│   └── 胜负判定        - 检查是否所有箱子都在目标点上
```

### 数据结构

#### 格子类型 (CellType)

```zen-c
enum CellType {
    Floor,   // 地板
    Wall,    // 墙壁
    Target,  // 目标点
}
```

#### 游戏状态 (SokobanGameState)

```zen-c
struct SokobanGameState {
    currentLevel: int,      // 当前关卡索引
    playerX: int,           // 玩家X坐标
    playerY: int,           // 玩家Y坐标
    steps: int,             // 步数
    levelWon: bool,         // 当前关卡是否获胜
    allWon: bool,           // 是否全部通关
    
    grid: [15][20]CellType, // 网格数据
    gridWidth: int,         // 网格宽度
    gridHeight: int,        // 网格高度
    
    boxes: [10]Point,       // 箱子位置
    boxCount: int,          // 箱子数量
    
    targets: [10]Point,     // 目标点位置
    targetCount: int,       // 目标点数量
}
```

## 渲染设计

### 颜色方案

| 元素 | 颜色 | RGB |
|------|------|-----|
| 墙壁 | 深灰 | (80, 80, 80) |
| 地板 | 深灰 | (45, 45, 50) |
| 目标点 | 绿色 | (0, 200, 100) |
| 箱子 | 棕色 | (180, 120, 60) |
| 箱子(在目标上) | 亮绿 | (0, 220, 120) |
| 玩家 | 蓝色 | (60, 140, 220) |
| 背景 | 深蓝黑 | (25, 25, 35) |

### 绘制顺序

1. 背景
2. 地板/墙壁格子
3. 目标点标记
4. 箱子
5. 玩家
6. UI 面板
7. 状态文字

## 碰撞检测

### 移动检测流程

1. 计算新位置 `(newX, newY)`
2. 检查新位置是否是墙 → 不能移动
3. 检查新位置是否有箱子
   - 有箱子：检查箱子后面 `(boxNewX, boxNewY)` 是否可移动
   - 箱子后面是墙或另一个箱子 → 不能移动
   - 否则：推动箱子，移动玩家
4. 无箱子：直接移动玩家

### 胜负判定

每移动一步后检查：
- 遍历所有目标点
- 检查每个目标点是否有箱子
- 所有目标点都有箱子 → 关卡完成

## 帧率控制

- 目标帧率：60 FPS
- 帧时间：1000/60 ≈ 16.67 ms
- 玩家输入通过事件驱动，不依赖帧率

## 错误处理

- 窗口创建失败：输出错误信息并退出
- 编译失败：显示具体错误信息
- 游戏运行中 ESC：正常退出

## 扩展性

### 添加新关卡

在 `sokoban_logic.zc` 中：

1. 在 `MAX_LEVELS` 中增加关卡数量
2. 添加新的关卡字符串常量
3. 在 `parseLevel` 中添加对新关卡的支持

### 修改格子大小

在 `main_window.zc` 中修改 `CELL_SIZE` 常量。
