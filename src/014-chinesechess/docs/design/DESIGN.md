# 中国象棋游戏设计文档

## 1. 概述

### 项目名称
Zen-C 中国象棋 (Chinese Chess)

### 项目目标
使用 Zen-C 语言和 003-framework 框架开发一款完整的中国象棋游戏，实现玩家 vs 电脑对战，红方先手。

### 核心功能
- 9x10 标准棋盘绘制
- 7种棋子正确移动规则
- 鼠标点击操作
- 简单AI对手

## 2. 技术规格

## 4. 移动规则

### 将/帅 (King)
- 移动范围: 宫内（3x3区域）
- 移动方式: 横竖走一步
- 约束: 不能出宫

### 仕/士 (Advisor)
- 移动范围: 宫内对角线
- 移动方式: 斜线走一步
- 约束: 不能出宫

### 相/象 (Elephant)
- 移动范围: 本方半场
- 移动方式: 斜线走两步（田字）
- 约束: 
  - 不能过河
  - 田字中心不能有棋子

### 马 (Horse)
- 移动方式: L形（日字）
- 约束: 蹩马腿（直线方向一格不能有棋子）

### 车 (Chariot)
- 移动范围: 任意距离
- 移动方式: 横竖直线
- 约束: 路径不能有其他棋子

### 炮 (Cannon)
- 移动范围: 任意距离
- 移动方式: 横竖直线
- 约束:
  - 移动时路径不能有棋子
  - 吃子时中间必须隔一个棋子（炮台）

### 兵/卒 (Pawn)
- 移动方式:
  - 未过河: 只能前进
  - 已过河: 可以前进或左右

## 5. AI 设计

### 评估函数
```zen-c
fn evaluateMove(game, pieceIndex, toCol, toRow) -> int {
    var score = 0;
    
    // 吃子得分
    if 目标有棋子 {
        match 棋子类型 {
            将/帅: 10000
            车: 900
            马: 400
            炮: 450
            仕/士/相: 200
            兵/卒: 100
        }
    }
    
    // 位置奖励
    if 兵已过河 { score += 50; }
    if 车在中线 { score += abs(col-4) * 5; }
    
    // 随机性
    score += rand(0, 20);
}
```

### AI 决策
1. 遍历所有黑方棋子
2. 枚举所有可能的目标位置
3. 使用评估函数打分
4. 选择最高分移动

## 9. 实现要点

### 坐标转换
- 屏幕坐标 → 棋盘坐标: `col = (x - BOARD_X) / CELL_SIZE`
- 棋盘坐标 → 屏幕坐标: `x = BOARD_X + col * CELL_SIZE`

### 路径检测
- 车/炮直线移动使用 Bresenham 算法检测路径
- 象/马的蹩腿检测独立判断

### AI 响应
- 使用定时器周期性触发 AI 移动
- 避免阻塞主线程
