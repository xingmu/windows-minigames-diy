# Badminton Game Design Document
# 088-Badminton 羽毛球游戏设计文档

## 1. 概述

### 游戏类型
- 体育类游戏 - 羽毛球对战

### 核心玩法
- 玩家控制左侧球拍与 AI 控制的右侧球拍进行羽毛球对战
- 先得 11 分者获胜

### 目标平台
- Windows 10/11 (主要)
- Linux/macOS (支持)

## 2. 技术架构

### 语言与框架
- **语言**: Zen-C
- **框架**: 003-framework
- **继承**: Game 基类

### 核心类
```
BadmintonGame extends Game
├── Player 数据 (x, y, width, height, score)
├── AI 数据 (x, y, width, height, score)
├── Shuttlecock 数据 (x, y, vx, vy, active)
└── 游戏状态 (gameStarted, gameOver, winner, servingPlayer)
```

## 3. 游戏规则

### 场地设置
- 场地尺寸: 800x600 像素
- 球场的左右边界: COURT_LEFT=50, COURT_RIGHT=750
- 球场的上下边界: COURT_TOP=50, COURT_BOTTOM=550
- 网的位置: NET_Y=300 (场地中间)

### 球拍规格
- 宽度: 60 像素
- 高度: 10 像素
- 移动速度: 8 像素/帧

### 羽毛球物理
- 大小: 8 像素半径
- 重力加速度: 0.3 像素/帧²
- 空气阻力: 0.99 (每帧速度衰减)
- 击球力度: 12.0
- 最大击球角度: 60 度

### 计分规则
- 球落在边界外，对方得分
- 球没过网落在本方半场，对方得分
- 先得 11 分获胜
- 得分后由得分方发球

### AI 行为
- 反应延迟: 80ms
- 失误范围: ±25 像素
- AI 只在球在本方半场或飞向本方时移动
- AI 有一定的预判能力但不是完美的

## 4. 操控

### 玩家控制
| 按键 | 功能 |
|------|------|
| W | 上移 |
| S | 下移 |
| A | 左移 (限制在左侧半场) |
| D | 右移 (限制在左侧半场) |
| 空格 | 发球 |
| ESC | 退出 |
| R | 重新开始 |

### 发球机制
- 游戏开始时玩家先发球
- 按空格键发球
- 发球后球有初始速度和角度

## 5. 视觉设计

### 颜色方案
- 球场背景: 绿色 (RGB: 50, 100, 50)
- 边界线: 白色
- 球拍: 玩家蓝色，AI 红色
- 羽毛球: 白色 cork，灰色羽毛
- 球网: 白色 + 棕色支架
- 计分: 对应球拍颜色

### UI 元素
- 顶部中央: 分数显示
- 左上角: 连击数 (rally count)
- 右上角: 发球方提示
- 场地界线和服务线

## 6. 物理碰撞

### 羽毛球碰撞检测
1. **边界碰撞**: 左右墙壁和上下边界
2. **球拍碰撞**: 矩形碰撞检测
3. **球网碰撞**: 如果球在网高度以下且未过网，则判定失误

### 碰撞响应
- 墙壁: 速度反转并衰减 0.8
- 天花板: 速度反转并衰减 0.7
- 球拍: 根据击中位置产生不同角度的反弹

## 7. 状态管理

### 游戏状态
```
Ready -> Playing -> Won/Lost
         ↑___________↓ (R键重置)
```

### 球拍状态
- 空闲状态
- 击球状态 (200ms 动画)
