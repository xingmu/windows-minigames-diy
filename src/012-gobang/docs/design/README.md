# Gobang 五子棋 - 设计文档

## 1. 概述

Zen-C 语言开发的经典五子棋游戏，采用 003-framework 框架，15x15 棋盘，玩家 VS 电脑 AI。

## 2. 技术架构

### 2.1 项目结构

```
012-gobang/
├── src/
│   ├── platform_api.zc   # 平台 API 定义
│   ├── game_logic.zc     # 游戏逻辑 + AI
│   └── main_window.zc    # 主程序 + 渲染
└── docs/design/
    └── README.md          # 本文档
```

### 2.2 核心模块

**game_logic.zc** - 棋盘状态管理（15x15 二维数组）、落子合法性检测、获胜检测（横/竖/斜四方向扫描）、AI 威胁评估算法

**main_window.zc** - 窗口创建与事件循环、帧缓冲渲染、棋盘绘制（网格、星位、棋子）、鼠标/键盘事件处理

## 3. 数据结构

### 棋盘状态

```
board: [15][15]int  // 0=空, 1=黑, 2=白
currentPlayer: int  // 当前玩家
gameOver: bool      // 游戏是否结束
winner: int         // 获胜方
```

### 获胜信息

```
WinInfo {
    hasWin: bool,
    winner: int,
    startRow/startCol: int,  // 获胜线起点
    endRow/endCol: int,      // 获胜线终点
}
```

## 4. AI 算法

### 4.1 威胁评估

对每个空位在四个方向进行评分：

| 棋型 | 英文 | 分数 |
|------|------|------|
| 五连 | Five | 100000 |
| 活四 | Open Four | 10000 |
| 冲四 | Blocked Four | 1000 |
| 活三 | Open Three | 800 |
| 眠三 | Sleep Three | 200 |
| 活二 | Open Two | 100 |
| 眠二 | Sleep Two | 50 |
| 活一 | Open One | 10 |

### 4.2 评分逻辑

每个空位得分 = sum(四个方向评分)

1. **连续己方棋子数**: 统计该方向上连续的己方棋子
2. **开放端数**: 统计该方向两端的空位数量
3. **阻挡情况**: 是否被对方棋子或边界阻挡

### 4.3 AI 决策

1. 第一手落在天元（中心点）
2. 同时评估己方进攻和对方防守
3. 若对方威胁更大（>1.2倍），优先防守
4. 五连/冲四优先响应

## 5. 渲染设计

### 5.1 窗口布局

```
[标题栏] 五子棋 Gobang    [状态]    ~55px
-------------------------------------------
                                     |
         15x15 棋盘                  |  ~600px
         (BOARD_OFFSET=30)           |
                                     |
-------------------------------------------
[状态栏] 操作提示                    |  ~40px

总计: 680 x 700 像素
```

### 5.2 棋盘参数

- 棋盘背景色: rgb(220, 179, 133) 木纹色
- 网格线: 黑色，1px
- 交叉点: CELL_SIZE = 40px
- 棋子半径: PIECE_RADIUS = 17px
- 星位（天元和四角星）: 半径 3px 圆点

### 5.3 棋子渲染

- 黑棋: rgb(10, 10, 10)，边缘 rgb(80, 80, 80)
- 白棋: rgb(245, 245, 245)，边缘 rgb(180, 180, 180)
- 最后落子: 额外一圈高亮环

## 6. 事件处理

### 6.1 鼠标事件

- 左键点击: 在有效交叉点落子
- 落子判定: 点击位置距离交叉点 <= CELL_SIZE * 3/5

### 6.2 键盘事件

- ESC: 退出游戏
- R: 重新开始

### 6.3 玩家回合控制

- 仅在 currentPlayer == BLACK（玩家回合）时响应鼠标
- 落子后自动触发 AI

## 7. 获胜检测算法

对给定位置，向四个方向（横、竖、左斜、右斜）扫描，统计连续相同棋子数，达到 5 即获胜。
