# LinkLink 游戏设计文档

## 1. 概述

LinkLink（连连看）是一款经典的益智游戏，玩家需要找出并消除所有配对的相同方块。

## 2. 游戏规格

### 2.1 游戏区域
- **网格尺寸**: 8×8 = 64 个方块
- **方块大小**: 48×48 像素
- **方块间距**: 4 像素
- **方块类型**: 18 种不同图案（每种出现2次，共32对）

### 2.2 游戏规则
1. 每个方块成对出现（相同图案）
2. 玩家点击两个相同的方块即可尝试消除
3. 两个方块必须能在**最多2个拐角**内连通才能消除
4. 连通路径上不能有其他方块阻挡

### 2.3 计分系统
- **基础分**: 每消除一对 +100 分
- **时间奖励**: 剩余时间(秒) / 10
- **路径奖励**: (3 - 拐角数) × 20
- **最终分数**: 基础分 + 时间奖励 + 路径奖励

### 2.4 时间限制
- 游戏时间: 5 分钟（300秒）
- 时间归零时游戏结束

## 3. 算法设计

### 3.1 连接路径检测

连连看的核心算法是判断两个方块是否能连通：

```
连通条件：
1. 直接连通：同一直线上，中间无阻碍
2. 一个拐角：需要1个转折点
3. 两个拐角：需要2个转折点
```

### 3.2 路径查找流程

```
function canConnect(r1, c1, r2, c2):
    // 1. 检查直接连通
    if canConnectDirect(r1, c1, r2, c2):
        return true
    
    // 2. 检查一个拐角
    if canConnectOneCorner(r1, c1, r2, c2):
        return true
    
    // 3. 检查两个拐角
    if canConnectTwoCorners(r1, c1, r2, c2):
        return true
    
    return false
```

### 3.3 直线连通检测

```
function canConnectDirect(r1, c1, r2, c2):
    if r1 != r2 and c1 != c2:
        return false  // 不在同一直线
    
    if r1 == r2:
        // 水平方向
        for c in (min(c1,c2) + 1) .. (max(c1,c2) - 1):
            if tiles[r1][c] is visible:
                return false
        return true
    
    // 垂直方向同理
```

### 3.4 边界处理

边界外的空白区域可以"绕行"，这增加了连接的可能性。

## 4. 数据结构

### 4.1 方块结构

```c
struct Tile {
    int type;      // 方块类型 (0-17)
    bool visible;  // 是否可见
}
```

### 4.2 游戏状态

```c
struct LinkLinkState {
    Tile[8][8] tiles;       // 方块网格
    int selectedRow;        // 选中行 (-1 = 无)
    int selectedCol;        // 选中列
    int score;              // 当前分数
    int timeLeft;           // 剩余时间
    bool gameOver;          // 游戏结束
    bool gameWon;           // 游戏胜利
    PathPoint[4] path;     // 连接路径
    int pathLength;         // 路径长度
}
```

## 5. 视觉设计

### 5.1 配色方案

| 元素 | 颜色 | RGB |
|------|------|-----|
| 背景 | 深蓝灰 | (30, 30, 40) |
| 方块背景 | 蓝灰 | (80, 80, 100) |
| 选中高亮 | 黄色 | (255, 255, 100) |
| 连接线 | 绿色 | (0, 255, 100) |
| 方块图案 | 调色板 | 12种颜色 |

### 5.2 方块设计

- **形状**: 圆角矩形（6px圆角）
- **边框**: 2px 高亮边框
- **图案**: 6种基础形状（圆、方、三角、菱形、星、十字）

## 6. 交互设计

### 6.1 鼠标交互

| 动作 | 响应 |
|------|------|
| 点击空位置 | 无响应 |
| 点击方块 | 选中/高亮方块 |
| 点击相同图案可连通 | 消除并显示连接线 |
| 点击相同图案不可连通 | 取消选中，高亮新方块 |
| 点击不同图案 | 取消选中，高亮新方块 |

### 6.2 键盘交互

| 按键 | 功能 |
|------|------|
| ESC | 退出游戏 |
| R | 重新开始 |

## 7. 技术实现

### 7.1 渲染方式

使用帧缓冲 (Framebuffer) 直接写入显存：
- 每个像素 32 位 (ARGB)
- Pitch = 宽度 × 4 字节
- 双缓冲消除闪烁

### 7.2 事件处理

- 鼠标事件：点击选择方块
- 键盘事件：R 重新开始，ESC 退出
- 定时器事件：每秒更新时间

### 7.3 游戏循环

```
while running:
    processEvents()    // 处理输入
    update()           // 更新逻辑
    render()           // 渲染画面
```

## 8. 扩展功能

### 8.1 提示功能
当玩家找不到配对时，可以提供提示。

### 8.2 重新排列
当没有可用移动时，自动重新排列方块。

### 8.3 道具系统
- 炸弹：消除一个方块
- 刷新：重新排列所有方块
