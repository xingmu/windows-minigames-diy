# Tower Defense - 游戏设计文档

## 1. 概述

Tower Defense 是一款经典的塔防游戏，玩家需要在地图上建造防御塔来阻止敌人到达终点。

## 2. 游戏设计

### 2.1 游戏区域

- 地图尺寸: 800 x 520 像素 (20 x 13 格子)
- UI 面板: 底部 80 像素
- 格子大小: 40 x 40 像素

### 2.2 路径设计

路径从地图左侧进入，经过多个转折点，最终到达底部左侧的终点。

路径点:
```
(0,2) -> (4,2) -> (4,5) -> (10,5) -> (10,8) -> (6,8) -> (6,10) -> (15,10) -> (15,6) -> (18,6) -> (18,12) -> (0,12)
```

### 2.3 防御塔

#### 箭塔 (Arrow Tower)
- 价格: 50 金币
- 伤害: 10
- 射程: 120 像素
- 攻击间隔: 500ms
- 特性: 快速攻击，单体伤害

#### 炮塔 (Cannon Tower)
- 价格: 100 金币
- 伤害: 30
- 射程: 100 像素
- 攻击间隔: 1200ms
- 特性: 范围伤害 (50像素范围)

#### 冰塔 (Ice Tower)
- 价格: 75 金币
- 伤害: 5
- 射程: 100 像素
- 攻击间隔: 800ms
- 特性: 减速敌人 50%，持续 2 秒

### 2.4 敌人类型

#### 普通敌人 (Basic)
- 血量: 50
- 速度: 60 像素/秒
- 奖励: 10 金币

#### 快速敌人 (Fast)
- 血量: 30
- 速度: 100 像素/秒
- 奖励: 15 金币

#### 坦克敌人 (Tank)
- 血量: 150
- 速度: 40 像素/秒
- 奖励: 25 金币

#### Boss敌人 (Boss)
- 血量: 500
- 速度: 30 像素/秒
- 奖励: 100 金币
- 出现条件: 第 5 波及之后

### 2.5 波次系统

- 总波次: 10
- 每波敌人数: 5 + 波次 * 3
- 生成间隔: 1500ms - 波次 * 50ms (最小 500ms)
- 波次奖励: 50 + 波次 * 10 金币

敌人血量倍率: 1.0 + (波次 - 1) * 0.2

## 3. 游戏流程

### 3.1 游戏状态

1. **Ready**: 等待玩家开始
2. **Playing**: 游戏中
3. **WaveComplete**: 波次完成
4. **GameOver**: 游戏失败 (生命值 <= 0)
5. **Victory**: 胜利 (完成 10 个波次)

### 3.2 游戏循环

```
初始化 -> Ready 状态
    |
    v
按空格开始 -> Playing 状态
    |
    v
生成敌人 -> 更新敌人位置 -> 检查碰撞 -> 塔攻击
    |
    v
敌人到达终点? -> 扣除生命
    |
    v
所有敌人死亡? -> 波次完成
    |
    v
所有波次完成? -> Victory
    |
    v
生命 <= 0? -> GameOver
```

## 4. 用户界面

### 4.1 UI 布局

```
+----------------------------------------+
|                                        |
|              游戏区域                   |
|           (800 x 520)                  |
|                                        |
+----------------------------------------+
| Gold: 200 | Lives: 20 | Wave: 1/10      |
| [Arrow 50] [Cannon 100] [Ice 75]       |
+----------------------------------------+
```

### 4.2 交互

- **建造**: 点击空地建造选中的防御塔
- **选择塔**: 点击底部按钮选择塔类型
- **快捷键**: 1/2/3 快速选择塔类型

## 5. 技术实现

### 5.1 架构

```
main_window.zc
    |
    +-- 游戏主循环
    +-- 事件处理 (键盘/鼠标)
    +-- 渲染管道
    |
    +-- 平台抽象层 (platform_api.zc)
    +-- Windows 实现 (platform_api_windows.zc)
```

### 5.2 核心数据结构

- `Tower`: 防御塔 (位置、类型、属性)
- `Enemy`: 敌人 (位置、血量、路径进度)
- `Projectile`: 投射物 (位置、目标、伤害)
- `PathPoint[]`: 路径点数组

### 5.3 关键算法

- **寻路**: 敌人沿预定义路径移动
- **目标选择**: 塔选择最近的敌人
- **碰撞检测**: 圆形碰撞 (投射物命中)
- **范围伤害**: 计算范围内所有敌人

## 6. 性能考虑

- 最大敌人数量: 30
- 最大塔数量: 20
- 最大投射物数量: 50
- 目标帧率: 60 FPS
