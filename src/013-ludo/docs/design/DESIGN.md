# 013-Ludo 飞行棋 - 游戏设计文档

## 1. 概述

### 游戏名称
Ludo 飞行棋

### 游戏类型
回合制棋盘游戏

### 核心功能
双人飞行棋游戏，玩家（红方）与电脑（蓝方）对战，目标是先将全部4枚棋子移动到终点。

### 目标用户
休闲游戏玩家，飞行棋爱好者

## 2. 游戏规则

### 2.1 基本规则

- **玩家数量**: 2人（玩家 vs 电脑）
- **棋子数量**: 每方 4 枚棋子
- **终点判定**: 最先将 4 枚棋子全部送达终点的一方获胜

### 2.2 骰子系统

- 每次回合开始时自动要求掷骰子
- 掷骰子结果为 1-6 的随机数
- **掷到 6**: 可以继续投掷，连续 3 次掷到 6 则强制换手
- **未掷到 6**: 回合结束，切换到对手

### 2.3 棋子移动

- **在家棋子**: 必须掷到 6 才能出动到起点
- **在轨迹棋子**: 按骰子点数前进相应格数
- **超过终点**: 不能移动，选择其他棋子（如果有）

### 2.4 吃子规则

- 落在对方棋子上（非安全格）时，吃掉对方棋子
- 被吃的棋子返回基地，变成在家状态
- **安全格**: 起点位置为安全格，在安全格不会被吃

### 2.5 轨迹设计

- 外圈: 52 格（每方 13 格起点范围）
- 回家路: 每方 5 格进入终点
- 总行程: 52 + 5 = 57 格

### 3.2 核心数据结构

```zen-c
// 棋子状态
enum PieceState {
    InHome,   // 在家里
    OnTrack,  // 在轨迹上
    InGoal,   // 已到达终点
}

// 单个棋子
struct Piece {
    state: PieceState;
    trackPos: int;   // 轨迹位置
    pixelX: int;     // 像素坐标
    pixelY: int;
    homeIndex: int;  // 家里索引
}

// 游戏状态
struct LudoGame {
    players: [2]Player;
    currentPlayer: int;

    diceRolled: bool;

```

### 3.3 关键算法

#### 轨迹位置计算
- 棋盘为十字形，外圈 52 格
- 每个玩家有独立的起点偏移
- 玩家 0 起点在轨迹位置 0
- 玩家 1 起点在轨迹位置 13

#### 电脑AI评估函数
```
score = 基础分 + 位置分 + 吃子加分 + 安全格加分
- 在家棋子掷6: 80分
- 快到终点: 90+分
- 能吃对手: +50分
- 在安全格: +10分
```

### 3.4 渲染流程

1. 清屏（深灰色背景）
2. 绘制棋盘（十字形轨迹）
3. 绘制所有棋子（按玩家颜色）
4. 绘制状态栏（当前玩家、回合、棋子状态）
5. 绘制骰子（如果已掷）
6. 游戏结束时绘制结束画面

## 5. 控制流程

### 5.1 游戏流程图

```
开始游戏
    |
    v
初始化游戏状态
    |
    v
+-> 玩家回合
|   |
|   +-> 按空格掷骰子
|   |
|   +-> 有可移动棋子?
|       |是           |否
|       v             v
|   等待选择棋子    切换玩家
|       |
|       +-> 鼠标点击选择棋子
|       |
|       +-> 移动棋子
|       |
|       +-> 掷到6? -> 继续回合
|           |
|           +-> 否 -> 切换玩家
|
+-> 电脑回合 (自动)
    |
    +-> 自动掷骰子
    |
    +-> AI选择最优棋子
    |
    +-> 移动棋子
    |
    +-> 掷到6? -> 继续回合
        |
        +-> 否 -> 切换到玩家回合
```

### 5.2 输入处理

- **空格键**: 掷骰子（仅当前玩家可操作时）
- **鼠标左键**: 选择棋子移动
- **ESC**: 退出游戏

## 6. 性能考虑

- 60 FPS 游戏循环
- 直接帧缓冲绘制，无中间层
- 简单字符绘制（8x8位图）
- 棋盘固定大小 450x450，无缩放
