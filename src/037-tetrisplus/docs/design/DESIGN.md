# TetrisPlus 设计文档

## 概述

TetrisPlus 是基于 003-framework 的增强版俄罗斯方块游戏，增加了更多形状、更快速度和高级游戏机制。

## 设计目标

1. **增加可玩性**: 12种不同形状提供更多组合可能
2. **提升挑战**: 更快的下落速度增加游戏难度
3. **视觉辅助**: 幽灵方块帮助精确放置
4. **分数激励**: 连击和Back-to-Back系统奖励技术

## 游戏机制

### 形状系统

| ID | 名称 | 格子数 | 颜色 | 旋转状态数 |
|----|------|--------|------|-----------|
| 0 | I | 4 | 青色 | 4 |

| 5 | J | 4 | 蓝色 | 4 |

| 7 | Big-I | 4 | 亮青色 | 4 |
| 8 | Small-2x2 | 4 | 金色 | 1 |
| 9 | Plus | 5 | 粉红色 | 1 |
| 10 | Big-L | 4 | 深蓝色 | 4 |
| 11 | Small-3 | 3 | 浅绿色 | 2 |

### 速度系统

```
初始速度: 600ms
每10行: 速度 -40ms
最小速度: 50ms
```

速度计算公式: `interval = max(50, 600 - speedLevel * 40)`

### 计分公式

```
基础分 = [100, 300, 500, 800][lines-1] × level
Tetris奖励: 基础分 × 1.5 + 500
Combo奖励: combo × 50 (combo > 1时)
```

## 数据结构

### 棋盘
```zen-c
board: [BOARD_HEIGHT][BOARD_WIDTH]int  // 0=空, 1-12=颜色索引
```

### 当前方块
```zen-c
struct CurrentPiece {
    pieceType: TetrominoType,
    x: int,           // 列位置
    y: int,           // 行位置
    rotation: int,    // 旋转状态 0-3
}
```

## 渲染流程

1. 清屏 (黑色背景)
2. 绘制棋盘网格
3. 绘制幽灵方块 (如果启用)
4. 绘制已锁定方块
5. 绘制当前下落方块
6. 绘制侧边栏 (分数、等级、下一个方块预览)
7. 绘制游戏结束画面 (如适用)

## 关键算法

### 碰撞检测
检查方块4个格子是否超出边界或与已锁定方块重叠。

### 墙壁踢
旋转失败时尝试左右偏移1-2格。

### 幽灵位置
从当前位置向下扫描直到碰撞。
